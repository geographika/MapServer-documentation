WEB¶

EMPTY [url]
URL zu der weitergeleitet wird wenn eine Abfrage fehlschlägt. Falls nicht definiert, wird der Wert von ERROR benutzt.
ERROR [url]
URL zu der weitergeleitet wird wenn ein Fehler auftritt. Falls nicht definiert, werden alte häßliche MapServer Fehlermeldungen ausgegeben.
FOOTER [filename]
Template das als Letztes benutzt werden soll, wenn alles Andere bereits gesendet wurde. Nur für Mehrfachergebnisse.
HEADER [filename]
Template das als Erstes benutzt wird, wenn noch nicht gesendet wurd. Nur für Mehrfachergebnisse.
IMAGEPATH [path]
Pfad zum temporären Verzeichnis in das temporäre Dateien und Bilder gespeichert werden. Muss für den Webserver beschreibbar sein und, je nach Plattform, mit einem / oder enden.
IMAGEURL [path]
Basis URL für IMAGEPATH. Diese wird ggf. ergänzt um IMAGEPATH um dem Webbrowser Zugriff auf die Bilddateien zu geben.
LOG [filename]
Dateiname der MapServer Logdatei. Muss für den Webserver beschreibbar sein.
MAXSCALEDENOM [double]

Größter Maßstab bis zu dem eine Karte ausgeliefert wird. Wird ein größerer Maßstab angefordert, wird die Karte mit dem hier definierten Maßstab ausgeliefert. Dies verhindert ungewünschtes herauszoomen. Es wird nur der Nenner des eigentlichen Maßstabwertes angegeben. Z.B. für eine Karte im Maßstab 1:24000 wird 24000 angegeben. Implementiert in MapServer 5.0, ersetzt veralteten MAXSCALE Parameter.

Veraltet ab Version 5.0.

MAXSCALE [double]

(veraltet) Ab MapServer Version 5.0 ist MAXSCALEDENOM zu benutzen. Größter Maßstab bis zu dem eine Karte ausgeliefert wird. MAXTEMPLATE [file|url]
Template das benutzt wird, wenn der größte Maßstab überschritten wird. Hilfreich bei verschachtelten Anwendungen.
METADATA

Hiermit können beliebige Daten als Wertepaare gespeichert werden. Dies wird mit OGC WMS benutzt um z.B. den Layertitel zu definieren. Es erlaubt auch mehr Flexibilität bei der Templateerstellung, da auf die hier definierten Werte über Template Tags zugegriffen werden kann.

Beispiel:

METADATA
    title "My layer title"
    author "Me!"
END
MINSCALEDENOM [double]

Kleinster Maßstab bis zu dem eine Karte ausgeliefert wird. Wird ein kleinerer Maßstab angefordert, wird die Karte mit dem hier definierten Maßstab ausgeliefert. Dies verhindert ungewünschtes hineinzoomen. Es wird nur der Nenner des eigentlichen Maßstabwertes angegeben. Z.B. für eine Karte im Maßstab 1:24000 wird 24000 angegeben. Implementiert in MapServer 5.0, ersetzt veralteten MINSCALE Parameter.

MINTEMPLATE
Template das benutzt wird, wenn der kleinste Maßstab unterschritten wird. Hilfreich bei verschachtelten Anwendungen.
OUTPUTFORMAT [mime-type]
Format des Abfrageergebnisses. Standard ist “text/html”. Dies ist noch experimentell, stattdessen sollte das OUTPUTFORMAT Objekt benutzt werden.
TEMPLATE [filename|url]
Template oder URL die benutzt wird um die Abfrageergebnisse darzustellen.
